### 1. 시계열 데이터 (Time Series Data)

> 시간의 흐름에 따라 일정한 간격(초, 분, 시간, 일, 월, 연 등)으로 측정된 연속적인 데이터
>   - 특정 시간 간격을 두고 연속적으로 관측된 값
>   - 시간을 독립 변수(시간 축, X축)로, 관측값을 종속 변수(Y축)로 설정해 분석할 수 있음


#### 핵심 구성 요소
*'언제'라는 맥락이 시계열 데이터의 핵심 구성 요소*

| 정의 | 설명 | 예시 | 시각적 표현 |
|---|---|---|---|
| 추세(Trend) | 장기적으로 증가/감소하는 경향성 존재 | 국내총생산(GDP), 인구 증가율 | 꾸준히 증가하거나 감소하는 선형 그래프 |
| 계절성(Seasonality) | 계절젹 요인의 영향을 받아 1년, 혹은 일정 기간 안에 반복적으로 나타나는 패턴 | 추석, 설 등 | 주기적인 변동을 나타내는 그래프 |
| 주기성(Cycle) | 정해지지 않은 빈도, 기간으로 일어나는 상승/하락 | 경기 변동 | 주기적으로 상승/하강을 반복하는 파형 그래프 |
| 불규칙 요인(Irregular) | 예측 불가능한 불규칙적인 움직임 | 전쟁, 홍수 등 | 불규칙한 변동을 나타내는 그래프 |

#### 사용 이유
> **사용 이유**: 시간에 따른 변화를 분석하여 미래를 예측하고, 불확실성을 줄이기 위해
>   - 시간에 따른 추세, 계절성, 주기적 변동을 파악하고 미래를 예측(Forecasting)
>   - 예시
>     - 주가의 일별 종가
>     - 기온의 시간별 변화
>     - 월별 매출액
>     - 심장 박동수의 초별 기록
>     - 웹사이트 방문자 수의 일별 변화


| 이유 | 설명 |
|---|---|
| 시간에 따른 변화 분석 | - 과거부터 현재까지의 데이터를 통해 시간에 따른 변화 파악 가능<br>- 예) 주식 시장의 변동성, 기온 변화, 트래픽 패턴 등 |
| 추세(Trend) 식별 | - 장기적인 데이터 흐름을 분석해 증가·감소 추세나 주기적인 패턴 파악 가능 |
| 계절성(Sesonality) 분석 | - 특정 시점마다 반복되는 패턴을 분석하여 계절별·주기적인 변동 파악 가능<br>- 예) 전력 소비량은 여름과 겨울에 높고, 소매업 매출은 연말에 증가하는 경향 |
| 이상치(Outlier) 감지 | - 특정 시점에서 발생하는 급격한 변화(급등·급락) 감지 가능<br>- 예) 금융 데이터에서의 주가 급락, 서버 로그 데이터에서의 비정상적인 접속량 증가 탐지 가능 |
| 이벤트 감지(Event Detection) | - 시간의 흐름에 따라 특정 사건(예: 자연재해, 소셜미디어 트렌드 급상승, 금융시장 충격)이 데이터에 미치는 영향 분석 가능 |


#### 특징
1. 시간 의존성
   - 시계열 데이터는 각 데이터 포인트가 시간의 흐름에 따라 연속적으로 기록된 형태
   - 이전 시점과 이후 시점의 관계 중요 -> 데이터를 다룰 때 시계열의 순서를 반드시 고려해야 함
3. 자기 상관성
   - 시간이 지남에 따라 이전 값들이 이후 값에 영향을 미치는 경향이 있음 => "자기 상관성(Autocorrelation)"
   - 예) 날씨 데이터: 오늘의 온도는 어제나 며칠 전의 온도와 관련 있을 가능성이 크다
5. 트렌드와 계절성
   - 시간의 흐름에 따라 상승/하락하는 장기적인 트렌드, 계절적 주기 패턴이 나타날 수 있음
   - 예) 판매 데이터: 계절에 따라 변동하거나 주기적인 패턴을 보일 수 있음


#### 사용 방법
> 날짜별 값을 생성한 후, 선 그래프로 시각화하고, 이동 평균을 적용해 추세를 분석

### 2. 시계열 데이터 분석이 시간 기반으로 이해하는 데 중요한 이유

> *현실 세계의 현상은 시간 속에서 발생하고 변화하기 때문*
>   - 언제 값이 증가했는지
>   - 어떤 시점 이후 변화가 생겼는지
>   - 반복되는 패턴이 있는지
>   - 앞으로 어떻게 변할지


> 시계열 데이터 분석의 목적: '과거가 미래에 어떤 영향을 주는가'에 대한 답을 찾는 것
>   - 과거를 이해하고
>   - 현재를 파악하며
>   - 미래를 예측하는

| 항목 | 설명 |
|---|---|
| 시간적 맥락 | 언제 일어났는가? |
| 패턴 발견 | 어떤 구조가 반복되는가? |
| 예측 | 앞으로 어떻게 될까? |
| 이상 탐지 | 뭔가 이상한 시점이 있는가? |
| 자기상관 | 과거 자신이 현재에 얼마나 영향을 미치는가? |
| 시차효과 | 원인이 일어났으니 결과는 언제 오는가? |
| 시의성 | 지금 행동해야 하는가? |

### 2-1. 중요한 이유: 시간적 맥락(Temporal Context)의 보존

> 시계열 데이터에서는 같은 값이라도 언제 발생했는지에 따라 의미가 달라짐
>   - '같은 숫자라도 언제 발생했느냐에 따라 다른 의미를 가짐'

#### 편의점 매출 예시
가정) 한 편의점의 하루 매출이 100만 원

- 시간 맥락 없이 볼 경우
  - '오늘도 100만 원을 팔았구나'
- 시간 맥락과 함께 볼 때
    ```text
        1월 (설 연휴 시즌):  100만 원 → 평소 대비 -40% (위기 신호, 연휴라 손님이 없음)
        7월 (평범한 화요일): 100만 원 → 평소 대비 +5%  (무난한 하루)
        12월 24일 (크리스마스 이브): 100만 원 → 평소 대비 -30% (기대 대비 저조)
    ```
  - 시간 맥락이 있을 경우 같은 100만 원이 '위기 신호'일 수도, '정상 운영'이 될 수도 있음

### 2-2. 중요한 이유: 패턴/구조 발견 - 분해(Decomposition)

> 데이터 안에 숨어 있는 주기성이나 계절성을 찾는 데 중요

시계열 데이터 -> 추세(trend) + 계절성(seasonal) + 주기성(cycle) + 노이즈(잔차, residual)로 분리 가능

#### 항공 승객 수 데이터

```bash
from statsmodels.tsa.seasonal import seasonal_decompose
from statsmodels.datasets import get_rdataset
import pandas as pd
import matplotlib.pyplot as plt # 데이터 시각화

# 데이터 불러오기
data = get_rdataset("AirPassengers")

# 데이터 프레임 추출
df = data.data

# 시간 컬럼 생성
df["Month"] = pd.date_range(start="1959-01", periods=len(df), freq="MS")

# DatetimeIndex 설정
df.set_index("Month", inplace=True)

# 시계열 분해
result = seasonal_decompose(df['value'], model='multiplicative', period=12)

# 그래프용 데이터프레임
plot_df = pd.DataFrame({
    "Original": df["value"],
    "Trend": result.trend,
    "Seasonal": result.seasonal * df["value"].mean(),
    "Residual": result.resid * 100
})

# 1959년 1년치만
plot_one_year = plot_df["1959-01":"1959-12"]

# 그래프 그리기
plt.figure(figsize=(8, 6))

plt.plot(plot_one_year.index.month, plot_one_year["Original"], marker="o", label="original")
plt.plot(plot_one_year.index.month, plot_one_year["Trend"], linewidth=2, label="trend")
plt.plot(plot_one_year.index.month, plot_one_year["Seasonal"], linestyle="--", label="seasonality")
plt.plot(plot_one_year.index.month, plot_one_year["Residual"], marker="o", linewidth=1, label="residual")

plt.xticks(range(1, 13))

plt.title("Time Series Data")
plt.xlabel("month")
plt.ylabel("passengers")

plt.grid(True)
plt.legend()

plt.show()
```

```text
시계열 데이터 불러오기
    ↓
날짜 인덱스 생성
    ↓
시계열 분해
    ↓
추세 / 계절성 / 잔차 추출
    ↓
그래프 시각화
```

### 2-3. 중요한 이유: 예측(Forecasting) 가능성

> 과거 패턴을 수식화해서 미래를 예측

#### 월별 승객 수라고 가정한 시계열 데이터 이용한 미래 예측
```bash
from statsmodels.tsa.arima.model import ARIMA
import pandas as pd

# 학습 데이터
train = pd.Series([112,118,132,129,121,135,148,148,136,119,104,118,
                   115,126,141,135,125,149,170,170,158,133,114,140])

# ARIMA(p=2, d=1, q=2) 모델 학습
# p: 자기회귀 차수, d: 차분 횟수, q: 이동평균 차수
model = ARIMA(train, order=(2,1,2))
fitted = model.fit()

# 6개월 미래 예측
forecast = fitted.forecast(steps=6)
conf_int = fitted.get_forecast(steps=6).conf_int(alpha=0.05)

# x축 생성
train_x = range(len(train))
forecast_x = range(len(train), len(train)+6)

# 그래프
plt.figure(figsize=(12,6))

# 원본 데이터
plt.plot(train_x, train, marker="o", label="train data")

# 예측값
plt.plot(forecast_x, forecast, marker="o", linestyle="--", label="forecast")

# 신뢰구간
plt.fill_between(forecast_x, conf_int.iloc[:,0], conf_int.iloc[:,1], alpha=0.2, label="95% confidence interval")

plt.xlabel("month")
plt.ylabel("passengers")

plt.grid(True)
plt.legend()

plt.show()
```

<details>
<summary>➕ ARIMA 모델</summary>

- 시계열 예측 모델
- AR + I + MA
  - AR(Auto Regressive): 자기회귀 -> 과거 값으로 현재를 예측
    ```text
      이번 달 승객 수 = 지난달 승객 수 + 2개월 전 승객 수
    ```
  - I(Integrated): 차분 -> 데이터의 추세 제거
    ```text
      100/110/120/130 => 10/10/10
    ```
  - MA(Moving Average): 이동평균 -> 과거 오차를 이용
    ```text
      예측이 얼마나 틀렸는가
    ```

</details>

```text
월별 승객 수라고 가정한 시계열 데이터 생성
    ↓
ARIMA 모델이 과거 24개월 패턴 학습
    ↓
앞으로 6개월 예측
    ↓
95% 신뢰구간 시각화(145~155 안에 있을 확률 95%)
```

### 2-4. 중요한 이유: 이상 탐지(Anomaly Detection)

> 정상 패턴을 정의하고, 그 범위를 벗어난 점을 이상치로 탐지함

#### 서버 트래픽 데이터 가정
```bash
import numpy as np
import pandas as pd

# 서버 트래픽 데이터
np.random.seed(42)
normal = np.random.normal(loc=500, scale=30, size=96)  # 정상 트래픽 (평균 500, 표준편차 30, 총 96개 시점)

# 이상 구간 주입: DDoS 공격 상황처럼 급격한 요청 증가
normal[40:45] = [1500, 2200, 2800, 2500, 1800]

# DataFrame 생성
df = pd.DataFrame({"time": range(len(normal)), "traffic": normal})

# 정상 구간 기준 평균과 표준편차 계산
mean = normal[:40].mean()
std = normal[:40].std()

# Z-score 계산
z_scores = np.abs((normal - mean) / std)

# Z-score가 3보다 큰 지점 탐지
anomalies = np.where(z_scores > 3)[0]

# 정상 범위
upper_bound = mean + 3 * std
lower_bound = mean - 3 * std

# 그래프 그리기
plt.figure(figsize=(12, 6))
plt.plot(df["time"], df["traffic"], marker="o", label="server traffic")
plt.scatter(anomalies, normal[anomalies], s=100, label="anomaly", color="red")

plt.axhline(upper_bound, linestyle="--", label="upper threshold", color='green')
plt.axhline(lower_bound, linestyle="--", label="lower threshold", color='green')

plt.xlabel("time")
plt.ylabel("requests per second")

plt.grid(True)
plt.legend()

plt.show()
```

```text
초당 약 500건의 요청을 받고 있다고 가정
    ↓
40-44번 시점까지 트래픽 크게 증가 (이상치)
    ↓
Z-score 계산 (어떤 값이 평균에서 얼마나 멀리 떨어져 있는지)
    ↓
Z-score가 3보다 크면 이상치로 판단
```

### 2-5. 중요한 이유: 자기상관(Autocorrelation) 분석

> 시계열 데이터는 자기 자신의 과거 값과 상관관계를 가짐 -> 이를 측정하면 몇 주기 전 값이 현재에 영향을 주는지 알 수 있음

#### 가상의 기온 데이터를 활용한 자기상관
```bash
from statsmodels.graphics.tsaplots import acf
import pandas as pd
import numpy as np
import matplotlib.pyplot as plt

# 재현성
np.random.seed(42)

# 365일 데이터 생성
days = np.arange(365)

temps = (
    15                                      # 평균 15도
    + 10 * np.sin(2 * np.pi * days / 30)    # 30일 주기 반복
    + np.random.normal(0, 2, 365)           # 노이즈 (평균 0, 표준편차 2의 랜덤오차)
)

# 시계열 데이터 변환
temps = pd.Series(temps)

# ACF 계산 (0~20까지 자기상관 계산)
acf_values = acf(
    temps,
    nlags=20
)

# 데이터프레임 생성
acf_df = pd.DataFrame({"lag": range(21), "acf": acf_values})

# 막대 그래프
plt.figure(figsize=(12, 6))

# lag별 자기상관계수를 막대로 표현
plt.bar(acf_df["lag"], acf_df["acf"])

# 상관관계 없음
plt.axhline(y=0, linewidth=1)

plt.title("ACF (Autocorrelation Function)")
plt.xlabel("Lag")
plt.ylabel("Autocorrelation")
plt.xticks(range(0, 21, 5))
plt.grid(True)

plt.show()
```

```text
가상의 기온 데이터 생성
    ↓
30일 주기 패턴 삽입 (30일 주기 패턴 + 랜덤 오차를 가진 시계열 데이터 생성)
    ↓
ACF 계산
    ↓
Lag별 자기상관 시각화
```

- lag = 1
  - 오늘 기온 <-> 어제 기온 상관관계
- lag = 5
  - 오늘 기온 <-> 5일 전 기온 상관관계

### 2-6. 중요한 이유: 인과관계 및 시차 효과(Lag Effect) 분석

> 원인과 결과 사이에 시간 지연이 있는 경우, 시계열 분석으로 그 지연 크기 정량화 가능
>   - 현실에서 원인과 결과는 동시에 발생하지 X
>   - **시차 효과(Lag Effect)**: 원인이 발생하고 나서 일정 시간이 지난 후에야 결과가 나타나는 현상

#### 마케팅 예시
```text
  1주차: 광고 집행 시작 → 매출 변화 없음
    ↓
  2주차: 브랜드 인지도 상승 → 매출 소폭 증가
    ↓
  3주차: 구매 고려 단계 → 매출 본격 증가
    ↓
  4~5주차: 입소문 효과 → 매출 정점 도달
    ↓
  6주차 이후: 광고 소진 → 매출 서서히 감소
```
  - 온라인 광고를 대규모로 집행했을 때, 매출 증가가 즉각적으로 나타나지 않는 경우가 많음
  - 시차 효과를 알면 몇 주 후의 반응을 기다리고, 다음 캠페인 타이밍 계산 가능

#### 시차 효과 분석의 가치
- '지금 아무 일도 없어 보이는 것'이 정상임을 수치로 증명 가능
- 지연의 크기를 알면 결과가 나타날 시점을 미리 계산해서 그 전에 행동 가능


### 2-7. 중요한 이유: 의사결정 시의성(Timeliness) 향상

> '언제 행동할 것인가'를 데이터 기반으로 결정 가능
>   - 과거 패턴에서 행동 타이밍의 신호 추출 가능

#### 농업 수확 타이밍 예시
```text
  활용 데이터:
    - 일조량 누적 데이터 (최근 30일)
    - 일교차 추이 (당도 상승과 직결)
    - 과거 수확일별 당도·품질 데이터 (5년치)
    - 현재 당도 측정치 추이 (일별)
  
  분석 결과:
    당도가 일주일 연속 상승하다가 상승 폭이 꺾이는 날로부터
    3일 이내가 최적 수확일 (이후는 산도 하락으로 품질 저하)
```
  - 수확 시점을 결정할 때 시계열 데이터 활용
  - 의사결정의 타이밍 최적화

### 3. 시계열 분석 없이 발생하는 문제

| 문제 유형 | 분석 없을 때 | 예시 |
|---|---|---|
| 허위 상관 | 무관한 두 변수를 인과관계로 오해 | '아이스크림 판매량 증가와 익사 사고 증가가 연관있다' -> '기온'이라는 공통 원인 때문 |
| 기저효과 | 특수 시점 비교를 일반적 변화로 착각 | '마케팅 성과로 12월 매출 증가' -> 연말, 계절적으로 당연히 오르는 달이므로 마케팅 성과와 상관 X |
| 계절성 | 계절 변화를 정책·행동의 효과로 오인 | '연말 헬스장 회원이 줄어 담당자 교체' -> 연말에 헬스장 회원이 감소하는 것은 전국적인 계절 패턴: 정상적인 흐름을 위기로 오인 |
| 지연 효과 | "효과 없음"이라고 조기 결론 | '담뱃값 인상 1개월 후 효과 없어 정책 폐기' -> 가격 인상의 효과는 단기보다 중장기에 나타남 |
| 추세 무시 | 성장인지 후퇴인지 구분 불가 | 'A사 쇼핑몰 매출이 작년 대비 20% 증가. 성장일까?' -> 전체 시장이 매년 25% 성장하는 추세라면 사실상 시장점유율이 하락한 것 |
| 단기 변동 오해 | 노이즈를 트렌드로 착각해 과잉 반응 | 'A 학생의 시험 성적이 75 -> 80 -> 85 -> 78 -> 90 일 때 4차에서 성적이 하락한 것인가' -> 전체 시계열로 보면 장기 우상향 추세 속 일시적 조정 |
