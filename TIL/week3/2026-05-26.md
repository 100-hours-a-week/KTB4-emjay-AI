### 1. Numpy

> 📢 대규모 다차원 배열 및 행렬 연산을 위한 고성능 수학 함수와 도구를 제공하는 파이썬 라이브러리 📢

| 브로드캐스팅 | 크기가 다른 배열 간의 연산을 자동으로 확장하여 반복문 없이도 효과적인 계산이 가능하도록 지원함 |
| 다양한 수학 함수 제공 | 선형대수, 통계, 난수 생성 등 수치 계산을 위한 다양한 기능을 포함하고 있음 |

### 1-1. 차원(Dimension)

> 📢 배열을 구성하는 축(axis)의 개수 📢
>   - 각 차원은 해당 배열의 특정 방향을 정의함
>   - 차원 증가 -> 데이터는 더 복잡한 구조 => 차원을 활용해 다차원 데이터를 효과적으로 처리 가능

| 구분 | 설명 |
|---|---|
| 차원과 축(Axis)의 관계 | 배열의 각 축(axis)은 특정 방향을 나타내며, 배열의 차원 수는 축의 개수와 동일함 |
| 차원의 확장(Expanding Dimension) | NumPy에서는 기존 배열에 새로운 차원을 추가할 수 있으며, 이를 통해 데이터를 보다 세분화하거나 연산을 최적화할 수 있음 |
| 차원 축소(Reducing Dimension) | 차원 축소는 데이터를 단순화하거나 특정 축을 따라 연산을 수행할 때 사용됨 |

| 차원 | 명칭 | 설명 | 예시 |
|---|---|---|---|
| 0차원 | 스칼라(Scalar) | - 단일 요소를 포함하는 배열<br>- 하나의 숫자나 문자를 의미함 | `np.array(42)` -> `42` |
| 1차원 | 벡터(Vector) | - 한의 축을 가짐<br>- 리스트 형태로 표현할 수 있는 배열 | `np.array([1, 2, 3])` -> `[1 2 3]` |
| 2차원 | 행렬(Matrix) | - 두 개의 축을 가짐<br>- 행(row)과 열(column)로 구성된 구조 | `np.array([[1, 2], [3, 4]])` -> `[[1 2] [3 4]]` |
| 3차원 | 텐서(Tensor) | - 세 개의 축을 가짐<br>- 여러 개의 2차원 행렬을 포함하는 구조 | `np.array([[[1, 2], [3, 4]], [[5, 6], [7, 8]]])` |
| N차원 | N차원 배열(N-Dimensional Array) | - 3차원 이상의 배열<br>- 여러 개의 축을 포함하는 복잡한 데이터 구조 | `np.array(...)` (n ≥ 4) |

> 📢 **사용 이유**: 데이터를 구조화하여 연산 속도 향상, 메모리 효율성, 수학적 일관성, 코드 간결성을 확보하기 위해
>   - 데이터를 조직적으로 관리하고 다양한 연산을 효율적으로 수행하기 위해 매우 중요한 개념
>   - 1차원 이상의 구조를 통해 데이터를 보다 체계적으로 구성/분석 가능

| 이유 | 설명 |
|---|---|
| 데이터의 조직화 및 표현 | NumPy의 차원 개념을 사용하면 데이터를 논리적으로 조작할 수 있으며, 1차원 리스트로 표현하기 어려운 복잡한 데이터도 손쉽게 구조화 가능 |
| 연산의 최적화 및 성능 향상 | 차원을 사용하면 연산을 특정 축을 기준으로 수행할 수 있어 계산 효율이 향상됨 |
| 대규모 데이터 처리 용이성 | 다차원 배열을 사용하면 이미지, 시계열 데이터 등 대량의 데이터를 효과적으로 다룰 수 있음 |
| 복잡한 문제 해결 가능 | 복잡한 데이터를 보다 논리적으로 다룰 수 있으며, 머신러닝 및 과학적 분석 작업에서 필수적인 요소로 작용함<br>다차원 배열을 통해 데이터 패턴을 쉽게 추출/분석 가능 |
| 코드의 가독성과 유지보수성 향상 | 데이터의 논리적 구조가 명확해지고, 인덱싱 및 슬라이싱이 직관적이어서 코드가 간결해지며 유지보수가 용이해짐 |

#### 📢 사용 방법

**1) 다양한 차원의 배열 생성 및 활용**
  - `.ndim` 속성을 통해 NumPy 배열의 차원 확인
  - 1. 0차원 배열(`스칼라`) 생성
      - 단일 요소를 포함하는 가장 단순한 형태의 배열
      - 하나의 숫자 or 문자와 같은 단일 데이터 저장에 사용됨
        ```bash
          import numpy as np
          scalar = np.array(42)
          print(scalar.ndim) # 출력: 0
        ```
    2. 1차원 배열(`벡터`) 생성
      - 리스트와 유사한 형태
        ```bash
          vector = np.array([1, 2, 3, 4, 5])
          print(vector.ndim)  # 출력: 1
        ```
    3. 2차원 배열(`행렬`) 생성
        ```bash
          matrix = np.array([[1, 2, 3], [4, 5, 6]])
          print(matrix.ndim) # 출력: 2
        ```
    5. 3차원 배열(`텐서`) 생성
        ```bash
          tensor = np.array([[[1, 2], [3, 4]], [[5, 6], [7, 8]]])
          print(tensor.ndim) # 출력: 3
        ```

**2) 차원 변경**
  - 1. `reshape()`를 통한 차원 변경
      - 배열의 크기를 유지하면서 차원 변경
      - 데이터의 총 개수는 변경 불가
      - 기존 데이터를 새로운 모양으로 배치하는 데 사용됨
        ```bash
          array = np.array([1, 2, 3, 4, 5, 6]) 
          print("array: ", array) 
          print("array.ndim: ", array.ndim) # 출력: 1
          reshaped_array = array.reshape(2, 3) # (2행, 3열)
          print("reshaped_array: ", reshaped_array) # 출력: [[1 2 3] [4 5 6]]
          print("reshaped_array.ndim: ", reshaped_array.ndim) # 출력: 2
        ```
    2. `newaxis`를 통한 차원 추가
      - 차원을 추가해 데이터의 형식 변경 가능
      - 주로 데이터를 모델에 입력하거나 특정 연산을 수행할 때 활용됨
        ```bash
          import numpy as np

          arr = np.array([1, 2, 3])
          print(arr.shape)

          # 행 벡터로 변환
          row_vec = arr[np.newaxis, :] # 앞쪽에 새 축을 추가한 뒤, 기존 원소들을 그대로 유지
          print('row_vec: ', row_vec) # 출력: [[1 2 3]]

          # 열 벡터로 변환
          col_vec = arr[:, np.newaxis] 
          print(col_vec) # 출력 [[1]
                         #      [2]
                         #      [3]]
        ```

### 1-2. 형태(Shape)

> 📢 배열의 각 차원(axis)별 요소 개수를 나타내는 튜플(tuple) 📢
>   - 배열이 가지고 있는 구조적인 특성을 나타냄
>   - 배열이 몇 개의 차원을 가지고 있고, 각 차원에서 몇 개의 요소를 포함하는지를 정확하게 표현하는 역할

<details>
<summary>➕ Tuple</summary>

> 파이썬에서 여러 개의 값을 하나로 묶어 저장할 수 있는 자료형
>   - 리스트와 비슷하게 순서가 있는 집합
>   - 한 번 생성하면 요소를 변경하거나 추가/삭제할 수 없는 불변(immutable) 자료 구조

</details>

#### Shape의 구성요소
| 요소 | 설명 |
|---|---|
| 차원(Dimension) | 배열이 몇 개의 축(axis)을 가지는 지 |
| 각 차원의 크기 | 배열의 각 축에 존재하는 요소의 개수를 의미 |

**예제**
  - 1차원 배열 (벡터)
    ```bash
      [1, 2, 3, 4]
    ```
    - 차원 수: 1
    - 형태(Shape): `(4, )` -> 요소가 4개인 1차원 배열
  - 2차원 배열 (행렬)
    ```bash
      [
        [1, 2, 3],
        [4, 5, 6]
      ]
    ```
    - 차원 수: 2
    - 형태(Shape): `(2, 3)` -> 2개의 행과 3개의 열을 가진 배열
  - 3차원 배열 (텐서)
    ```bash
      [  
        [    
      		 [1, 2], [3, 4]
        ],
        [  
      		 [5, 6], [7, 8]
        ]
      ]
    ```
    - 차원 수: 3
    - 형태(Shape): `(2, 2, 2)` -> 2개의 블록, 각 블록에 2개의 행, 각 행에 2개의 열

<details>
<summary>➕ 텐서</summary>

> 다차원 배열(행렬)을 일반화한 개념. 숫자 데이터를 다양한 차원으로 표현하는 수학적 구조
>   - 0차원: 스칼라 / 1차원: 벡터 / 2차원: 행렬 / 그 이상: 고차원 텐서
>   - 머신러닝, 딥러닝에서 데이터를 저장/연산하는 데 사용됨

</details>

#### Shape의 핵심 포인트
| 형태 | 내용 |
|---|---|
| 튜플로 표현 | - 배열의 형태는 불변하는 튜플 형식<br>- 각 차원의 크기가 순서대로 나열됨<br>- 예: `(m, n)`, `(x, y, z)` |
| 각 차원별 요소 개수 확인 | - 다차원 배열에서 각 축의 요소 개수를 빠르게 확인 가능<br>- 예: 3차원 배열의 형태가 `(4, 3, 2)`라면, 첫 번째 차원에는 4개의 배열이, 두 번째 차원에는 3개의 배열, 세 번째 차원에는 2개의 요소가 포함됨 |
| 배열 조작 시 중요한 속성 | 배열을 변경하거나 연산을 수행할 때 형태 정보를 활용해 배열의 일관성을 유지할 수 있음<br>- 예: `reshape()`를 사용해 배열의 형태를 변경할 때, 원소의 개수가 유지되어야 함 |

#### 다양한 차원의 배열 형태 예시
| 배열 형태 | 예제 코드 | 결과(`.shape`) | 설명 |
|---|---|---|---|
| 1차원 배열 | `np.array([1, 2, 3])` | `(3,)` | 1차원 배열<br>요소 3개 |
| 2차원 배열 | `np.array([[1, 2], [3, 4]])` | `(2, 2)` | 2차원 배열<br>2개의 행과 2개의 열 |
| 3차원 배열 | `np.array([[[1], [2]], [[3], [4]]])` | `(2, 2, 1)` | 3차원 배열<br>2개의 행, 2개의 열, 각 1개 요소 |
| 0차원 배열 | `np.array(5)` | `()` | 스칼라 값으로 차원이 없음 |

#### 형태(Shape)와 차원(Dimension)의 차이점
| 개념 | 설명 | 예제 | 
|---|---|---|
| 차원(ndim) | 배열이 가지고 있는 축(axis)의 개수를 의미 | `array.ndim → 2` |
| 형태(shape) | 각 차원별 요소의 개수를 튜플로 나타냄 | `array.shape → (2, 3)` |


> 📢 **사용 이유**: 배열 구조를 명확히 하여 연산의 일관성과 정확성을 보장하기 위해
>   - NumPy 배열은 다차원 데이터를 다룰 수 있는 기능을 제공하지만, 배열이 어떤 형태로 구성되어 있는지를 명확하게 파악하지 않으면 데이터 조작 및 분석 과정에서 오류 발생 가능
>   - 데이터를 올바르게 구조화하고, 연산 수행 시 불필요한 메모리 사용 방지

| 이유 | 설명 |
|---|---|
| 데이터의 체계적인 구조화 | NumPy의 형태를 사용하면 데이터를 논리적으로 조작하고, 복잡한 다차원 데이터를 일관된 방식으로 표현 가능 |
| 연산의 효율성 향상 | - 형태 정보를 활용하면 특정 차원을 기준으로 연산을 최적화 가능<br>- 예) 배열의 축(axis)을 지정하여 연산을 수행하면 반복문 없이도 데이터를 빠르게 처리 가능 |
| 메모리 사용 최적화 | - 배열의 형태를 통해 불필요한 데이터 복사를 방지하고, 메모리 공간을 효율적으로 활용 가능<br>- 특히, 대용량 데이터를 처리할 때 메모리 사용량을 줄여 성능 향상 가능 |

#### 📢 사용 방법

**1) 배열의 형태 확인**
  - `.shape` 속성을 통해 배열의 형태 확인
    ```bash
      import numpy as np
  
      # 2차원 배열 생성
      array = np.array([[1, 2, 3], [4, 5, 6]])
      
      # 배열의 형태 확인
      print(array.shape) # 출력: (2, 3)
    ```

**2) 배열의 형태 변경**
  - 형태 변경을 통해 데이터를 원하는 구조로 조정하여 분석/처리 작업을 효율적으로 수행 가능
  - 1. `reshape()` 사용한 형태 변경
      - 기존 데이터를 유지한 채 새로운 형태로 배열 변경 가능
      - 새로운 형태의 배열 크기는 원래 배열의 크기와 같아야 함
    ```bash
      array = np.array([1, 2, 3, 4, 5, 6])
      
      reshaped_array = array.reshape(2, 3)
      print(reshaped_array)  # 출력: [[1 2 3]
                             #       [4 5 6]]
    ```
  - 2. `resize()`를 사용한 형태 변경
      - 원본 배열 자체를 변경함
      - 필요한 경우 배열을 확장하거나 축소함 
    ```bash
      array = np.array([1, 2, 3, 4])
      array.resize(2, 3)
      print(array) # 출력: [[1 2 3]
                   #       [4 0 0 ]]
    ```
  - 3. `np.newaxis`를 사용한 차원 추가
      - 배열에 새로운 차원 추가 가능
    ```bash
      array = np.array([1, 2, 3])
      print('array: ', array)
      print('array.shape: ', array.shape)  # 출력: (3,) -> 1차원
      
      expanded_array = array[:, np.newaxis]
      print('expanded_array: ', expanded_array)
      print('expanded_array.shape: ',expanded_array.shape)  # 출력: (3, 1) -> 2차원
    ```
  - 4. `flatten()`을 사용한 다차원 배열 평탄화
       - 새로운 배열을 생성하며, 원본 배열에는 영향을 주지 않음
    ```bash
      array = np.array([[1, 2, 3], [4, 5, 6]])
      flattened_array = array.flatten()
      print(flattened_array) # 출력: [1 2 3 4 5 6]
    ```
  - 5. `ravel()`을 사용한 다차원 배열 평탄화 (참조 반환)
       - 다차원 배열을 1차원으로 변환하지만, 원본 배열에 대한 참조(view)를 반환함
    ```bash
      array = np.array([[1, 2, 3], [4, 5, 6]]) 
      print('array : ', array) # 출력: [[1 2 3] [4 5 6]]
      raveled_array = array.ravel()
      print('raveled_array: ', raveled_array) # 출력: [1 2 3 4 5 6]
      
      # 참조를 통한 원본 변경 확인
      raveled_array[0] = 99  
      print('[0]요소 변경후 array: ', array) # 출력: [[99  2  3] [ 4  5  6]]
    ```
  - 6. transpose()를 사용한 배열 축 전환
       - 배열의 차원을 전환해 데이터를 새로운 방식으로 재배치할 수 있도록 함
       - 행과 열을 바꾸거나, 다차원 배열의 축 순서를 변경 가능
    ```bash
      array = np.array([[1, 2, 3], [4, 5, 6]])
      transposed_array = array.transpose()
      print(transposed_array)  # 출력: [[1 4]
                               #       [2 5]
                               #       [3 6]]
    ```

### 1-3. 데이터 타입(Data Type)

> 📢 배열의 각 요소가 가질 수 있는 값의 유형을 지정하는 속성 📢

#### NumPy의 데이터 타입
| 데이터 타입 | 설명 | 예시 |
|---|---|---|
| 정수형(Integers) | 부호 있는 또는 없는 정수를 저장하는 타입<br>뒤의 숫자는 부동소수점을 저장하는 데 쓰이는 비트 수를 의미함 | `int8`, `int16`, `int32`, `int64`, `uint8`, `uint16` |
| 부동소수점형(Floating Point) | 소수점을 포함한 숫자를 저장하는 타입 | `float16`, `float32`, `float64` |
| 복소수형(Complex Numbers) | 실수와 허수를 포함한 복소수를 저장하는 타입 | `complex64`, `complex128` |
| 문자열형(Strings) | 고정된 길이의 문자열을 저장하는 타입 | `str_` 또는 `unicode_` |
| 불리언형(Boolean) | `True` 또는 `False` 값을 저장하는 타입 | `bool_` |
| 객체형(Object) | 파이썬 객체를 포함할 수 있는 타입 | `object_` |
| 유니버셜 타입(Generic) | 특정 크기에 국한되지 않는 범용 타입 | `int_`, `float_` |

#### NumPy 데이터 타입 특징
- C 언어의 기본 데이터 타입을 기반으로 함
- 일반적인 파이썬의 데이터 타입보다 메모리 사용량이 적고, 연산 속도가 빠른 것이 특징

<details>
<summary>❓ C언어</summary>

NumPy는 내부적으로 C언어로 구현됨 -> 파이썬의 기본 데이터 구조보다 연산 속도가 빠르고 메모리 효율이 높음
  - C언어의 정적 타입 시스템을 활용해 배열의 크기와 데이터 타입을 명확하게 지정 가능 -> 대량의 데이터를 빠르게 처리 가능
  - C언어 타입을 기반으로 하여 메모리 효율적인 처리 가능

</details>

| 특징 | 설명 |
|---|---|
| 정확한 크기 지정 | - NumPy는 각 데이터 타입의 크기를 명확하게 정의할 수 있도록 비트(bit) 단위의 크기를 지정 가능<br>- 예를 들어, int32는 32비트 크기의 정수를 의미함 |
| 동일한 타입 요소 유지 | NumPy 배열은 모든 요소가 동일한 데이터 타입을 가져야 함 |
| 자동 형변환 기능 | - 배열을 생성할 때, NumPy는 입력된 데이터에 따라 적절한 데이터 타입을 자동으로 설정함<br>- 그러나 명시적 형 변환을 통해 원하는 타입으로 변경도 가능 |


> 📢 **사용 이유**: 메모리를 절약하고 연산 속도를 높이기 위해
>   - 데이터 타입을 명확하게 정의하면 메모리 효율성, 연산 속도 향상, 데이터의 일관성 유지, 시스템 간 호환성 등 다양한 이점을 얻을 수 있음

| 이유 | 설명 |
|---|---|
| 메모리 효율성 향상 | - NumPy 배열은 각 요소에 대해 고정된 크기의 메모리를 할당하므로, Python의 동적 리스트보다 메모리를 적게 사용함<br>- 예) int8 타입을 사용하면 각 요소가 1바이트만 차지하므로 대량의 데이터를 효율적으로 저장 가능 |
| 연산 속도 향상 | - NumPy는 배열의 요소가 동일한 데이터 타입을 가지므로, 내부적으로 최적화된 C 기반 연산을 수행하여 Python의 기본 리스트보다 훨씬 빠른 연산 속도를 제공<br>- 대규모 데이터 분석 및 머신러닝 연산 시 성능이 중요한 요소 |
| 데이터 일관성 유지 | - 모든 요소가 동일한 데이터 타입을 가지므로, 연산 중 불필요한 형 변환이 발생하지 않으며, 일관된 데이터 처리 및 안정적인 계산이 가능<br>- 데이터 무결성을 보장하는 데 중요한 역할 |
| 시스템 및 라이브러리 간 호환성 | - NumPy의 데이터 타입은 C, Fortran과 같은 저수준 언어와 호환되도록 설계되어 있어, 다른 시스템 및 라이브러리와 데이터를 주고받을 때 효율적<br>- 머신러닝, 시뮬레이션, 데이터 분석 등 다양한 분야에서 필수적인 요소 |


#### 📢 사용 방법
- `.dtype` 속성을 통해 배열의 데이터 타입을 확인
- 배열 생성 시 `dtype` 인수로 특정 데이터 타입을 명시적으로 지정
- `astype()` 메서드 사용하여 기존 배열의 데이터 타입을 다른 타입으로 변환

<details>
<summary>💡 데이터 타입 변환 시 주의할 점</summary>

- 데이터 타입 변환 시
  - `정수 -> 부동소수점 변환`: 정밀도 증가
  - `부동소수점 -> 정수 변환`: 소수점이 절삭될 수 있음
- 값의 크기
  - `작은 크기의 데이터 타입으로 변환`: 메모리 절약 가능
  - `값의 범위를 벗어날 시`: 오버플로우(overflow) 발생 가능

| 기능 | 설명 | 예제 코드 |
|---|---|---|
| `astype()` | 명시적으로 데이터 타입 변환 | `array.astype(np.float32)` |
| `np.int32()` | 정수형으로 변환 | `np.int32(array)` |
| `np.float64()` | 부동소수점형으로 변환 | `np.float64(array)` |

</details>


### 1-4. 인덱싱(Indexing)

> 📢 배열 내 특정 요소에 접근하거나 값을 참조하기 위한 방법 📢
>   - 배열의 개별 요소뿐만 아니라 특정 범위의 요소에도 접근할 수 있으며, 데이터를 효과적으로 조회/조작 가능

**NumPy Indexing 종류**
| 종류 | 설명 |
|---|---|
| 정수(Indexing by Integer) | 개별 요소에 접근할 때 사용되는 가장 기본적인 인덱싱 방<br>배열의 특정 위치를 지정하여 값을 가져오거나 수정 가능 |
| 슬라이싱(Slicing) | 특정 범위의 요소를 선택할 때 사용되며, `start:end:step` 형식으로 지정<br>원하는 부분만 추출하거나 일정 간격의 데이터 선택 가능 |
| 불리언(Boolean) 인덱싱 | 조건을 만족하는 요소만 선택가능<br>조건식을 사용하여 필터링할 때 유용 |
| 팬시(Fancy) 인덱싱 | 리스트나 배열을 이용하여 특정 인덱스를 지정하여 여러 요소를 한 번에 선택 가능 |

**인덱싱 시 주의할 점**
- 인덱스 범위를 초과하면 오류 발생
  - 존재하지 않는 인덱스 참조할 시 `IndexError` 발생
- 슬라이싱 시 경계값 포함 여부
  - `start:end` 슬라이싱 시 `start`는 포함되지만 `end`는 포함 X
- 음수 인덱스 사용
  - 배열의 끝에서부터 요소를 접근할 때 음수 인덱스 사용 가능

> 📢 **사용 이유**: 배열의 특정 요소에 빠르게 접근하여 데이터를 선택적으로 조작하고 처리하기 위해
>   - 복잡한 데이터 조작을 쉽게 수행 가능
>   - 대용량 데이터를 처리할 때 메모리 효율성과 연산 속도 극대화 가능

#### 📢 사용 방법
- 정수, 슬라이싱, 불리언, 팬시 인덱싱을 사용해 특정 요소에 접근하거나 수정


### 1-5. 연산(Operation)

> 📢 배열의 요소에 대해 수행되는 수학적/논리적 계산 📢

#### NumPy 연산 주요 특징
- 벡터화(Vectorization)를 지원하여 반복문 없이도 빠른 연산 수행 가능
- 크기가 다른 배열 간의 연산을 자동으로 조정하는 브로드캐스팅(Broadcasting) 기능을 제공해 유연한 데이터 처리 가능

| 특징 | 설명 |
|---|---|
| 벡터화(Vectorization) 지원 | 반복문 없이 배열의 모든 요소에 대해 동시에 연산을 수행하므로 코드가 간결하고 실행 속도 빠름<br>예) c = a + b처럼 요소별 덧셈이 자동으로 한 번에 실행 |
| 요소별 연산(Element-wise Operation) | 배열의 각 요소에 대해 동일한 연산이 적용되며, 스칼라 값 또는 다른 배열과의 연산 가능 |
| 다양한 연산 유형 제공 | 기본적인 사칙연산(+, -, *, /)부터, 삼각함수, 지수/로그 연산, 비교 및 논리 연산 등 다양한 기능 제공 |
| 브로드캐스팅(Broadcasting) 기능 | 크기가 다른 배열 간의 연산을 자동으로 조정하여 크기를 맞춰 연산할 수 있도록 함 |
| 배열 축(axis) 기반 연산 | 특정 차원(행 또는 열)에 대한 합계, 평균 등 집계 연산을 지원 |

#### NumPy 연산의 종류
| 연산 유형 | 설명 | 예시 함수 |
|---|---|---|
| 산술 연산<br>(Arithmetic Operations) | 배열의 요소별 사칙연산 수행 | `+`, `-`, `*`, `/`, `np.add()`, `np.subtract()` |
| 비교 연산<br>(Comparison Operations) | 요소 간 크기 비교 및 불리언 값 반환 | `>`, `<`, `==`, `np.equal()`, `np.greater()` |
| 논리 연산<br>(Logical Operations) | 불리언 배열을 대상으로 AND, OR 연산 수행 | `np.logical_and()`, `np.logical_or()` |
| 통계 연산<br>(Statistical Operations) | 배열의 평균, 최댓값, 최솟값 등 통계적 계산 수행 | `np.mean()`, `np.sum()`, `np.min()`, `np.max()` |
| 선형대수 연산<br>(Linear Algebra Operations) | 행렬 곱, 전치, 행렬식 등 선형대수 계산 | `np.dot()`, `np.linalg.inv()`, `np.transpose()` |
| 브로드캐스팅<br>(Broadcasting Operations) | 크기가 다른 배열 간의 연산 지원 | 자동 배열 크기 조정 |

#### NumPy 연산의 동작 방식
| 동작 방식 | 설명 |
|---|---|
| 요소별(Element-wise) 연산 | NumPy 배열의 기본 연산은 요소별로 수행되며, 배열의 크기가 동일해야 함 |
| 브로드캐스팅(Broadcasting) | 크기가 다른 배열 간 연산 시 자동으로 차원이 확장되어 연산이 가능하도록 지원됨 |
| 인덱싱 및 슬라이싱과의 조합 | 배열의 특정 부분을 선택한 후 연산을 수행할 수 있음 |


> 📢 **사용 이유**: 대량의 데이터를 빠르고 정확하게 처리하여 분석/계산의 정확성을 높이기 위해
>   - 대용량 데이터 처리 효율성: 배열 기반의 연산을 통해 수백만 개 이상의 대용량 데이터 빠르게 연산 가능
>   - 다차원 데이터 연산 지원: 특정 축(axis)을 기준으로 연산이 가능해 데이터 분석에 유용함
>   - 브로드캐스팅을 통한 유연한 연산: 크기가 다른 배열 간 연산이 가능하도록 자동으로 배열 크기를 조정하는 브로드캐스팅 기능 지원
>   - 다양한 연산 기능 제공: 사칙연산 뿐만 아니라, 지수/로그 연산, 삼각 함수, 선형대수 연산 등 다양한 기능 제공

#### 사용 방법
- 배열의 각 요소에 대해 수학적, 논리적 연산을 수행
1. 요소별 연산 (Element-wise Operations)
    - 배열 크기 같은 경우 -> 요소별로 연산 수행
    - 배열 크기 다른 경우 -> 브로드캐스팅(Broadcasting) 규칙 적용하여 연산 수행
2. 비교 연산 (Comparison Operations)
    - 배열의 각 요소를 조건과 비교해 T/F 값 반환
    - 특정 조건을 만족하는 데이터를 쉽게 판별 가능
3. 통계 연산 (Statistical Operations)
    - 데이터의 분포를 요약하거나 특징을 수치로 나타내는 연산
    - 데이터의 중심(평균, 중앙값)과 범위(최댓값, 최솟값), 분산 정도(표준편차)를 빠르게 파악 가능
    - 탐색적 분석(EDA)에 활용됨
4. 선형대수 연산 (Linear Algebra Operations)
    - 벡터/행렬 같은 수학적 구조를 다룸
    - 사용 이유
      - 연산 속도: C 기반으로 구현된 함수를 사용해, 대규모 데이터의 선형대수 연산 빠르게 처리 가능
      - 수학적 활용성: 전치, 내적, 역행렬 같은 연산은 회귀분석, 신호 처리, 머신러닝 알고리즘 등에서 핵심적으로 사용
      - 코드 가독성: 수학적 기호를 그대로 옮긴 듯한 함수 이름 사용 -> 복잡한 연산을 짧고 명확하게 표현 가능

### 1-6. 유니버셜 함수(Universal Function)

> 📢 배열의 각 요소에 대해 반복적으로 수행되는 벡터화된 연산을 제공하는 함수 📢
>   - 단일 입력(single input) 및 다중 입력(multiple input)을 지원
>   - 요소별(element-wise)로 동작하여 효과적인 수학/논리 연산을 수행함

| 특징 | 설명 |
|---|---|
| 다양한 함수 제공 | 수학 함수(삼각함수, 지수, 로그), 논리 함수, 비교 함수 등 여러 가지 기본 및 고급 연산 포함 |
| 다중 입력 및 출력 지원 | 하나의 유니버설 함수는 여러 개의 입력 배열을 받아 연산을 수행하고, 하나 이상의 출력 반환 가능 |
| 유형 캐스팅(Type Casting) 지원 | 서로 다른 데이터 유형 간 연산을 지원하며, 필요에 따라 자동 형변환 수행 |
| 배열 브로드캐스팅 지원 | 크기가 다른 배열 간의 연산을 가능하게 하여 유연한 연산 처리 가능 |

#### 유니버셜 함수의 분류

| 유형 | 설명 | 예제 함수 |
|---|---|---|
| 산술 연산(Arithmetic Functions) | 사칙연산 및 수학적 연산 수행 | `np.add()`, `np.subtract()`, `np.multiply()`, `np.divide()` |
| 삼각 함수(Trigonometric Functions) | 삼각 및 역삼각 함수 계산 | `np.sin()`, `np.cos()`, `np.tan()`, `np.arcsin()` |
| 지수 및 로그 함수(Exponential and Logarithmic Functions) | 지수 및 로그 연산 수행 | `np.exp()`, `np.log()`, `np.log10()` |
| 비교 연산(Comparison Functions) | 배열 요소 간 비교 연산 수행 | `np.greater()`, `np.less()`, `np.equal()` |
| 논리 연산(Logical Functions) | 불리언 논리 연산 수행 | `np.logical_and()`, `np.logical_or()`, `np.logical_not()` |
| 비트 연산(Bitwise Functions) | 이진수 기반 비트 연산 수행 | `np.bitwise_and()`, `np.bitwise_or()`, `np.bitwise_xor()` |


> 📢 **사용 이유**: 반복문 없이 배열의 요소별 연산을 자동화하여 계산 속도를 높이고 메모리 사용을 최적화하기 위해
>   - 고속 연산 수행: C 기반으로 구현되어 있어 파이썬의 기본 반복문보다 훨씬 빠르게 연산 수행 가능
>   - 벡터화로 인한 코드 간결화: 반복문을 사용하지 않고 배열의 각 요소에 대해 연산 적용 가능 -> 코드 간결 & 가독성 향상
>   - 다양한 수학적 기능 제공: 사칙연산, 삼각함수, 지수, 로그 연산 등 다양한 수학적/논리적 연산을 한 번에 수행 가능
>   - 메모리 효율성: 브로드캐스팅 기능을 통해 크기가 다른 배열 간 연산을 자동으로 조정하여 메모리 절약 & 성능 최적화 가능
>   - 데이터 분석 및 과학적 계산 최적화: 벡터 연산을 통해 빠른 통계 분석 및 수치 계산 가능 -> 실시간 분석에 적합

#### 사용 방법
- 기본 문법
  - `np.함수명(배열, 옵션)` 형식으로 사용
  - `out` 매개변수를 사용하면 결과를 기존 배열에 저장
1. 산술 연산 (Arithmetic Operations)
   - 배열 전체에 대해 수학적 연산을 반복문 없이 수행 가능
   ```bash
      # np.add(): 요소별 덧셈
      np.add(a, b)
      
      # np.subtract(): 요소별 뺄셈
      np.subtract(a, b)
      
      # np.multiply(): 요소별 곱셈
      np.multiply(a, b)
      
      # np.divide(): 요소별 나눗셈
      np.divide(a, b)
      
      # np.power(): 거듭제곱 연산
      np.power(a, 2)
   ```
2. 비교 연산 (Comparison Operations)
   - 조건 판별을 통해 원하는 데이터를 빠르게 걸러낼 수 있음
   - 불리언 배열을 생성하여 조건에 맞는 요소를 선택 가능
   ```bash
      # np.equal(): 각 요소가 같은지 비교
      np.equal(a, b)
      
      # np.greater(): a의 요소가 b보다 큰지 비교
      np.greater(a, b)
      
      # np.less(): a의 요소가 b보다 작은지 비교
      np.less(a, b)
   ```
3. 논리 연산 (Logical Operations)
   - 불리언 배열의 각 요소에 대해 AND, OR, NOT 같은 논리 연산을 수행
   ```bash
      # np.logical_and(): 두 조건이 모두 참일 때 True
      np.logical_and(data > 15, data < 35)
      
      # np.logical_or(): 두 조건 중 하나라도 참일 때 True
      p.logical_or(data < 15, data > 35)
      
      # np.logical_not(): 조건이 거짓일 때 True
      np.logical_not(data > 20)
   ```
4. 삼각 함수 (Trigonometric Functions)
   - 각도를 입력으로 받아, 직각삼각형의 변의 길이 비율을 반환
   - 라디안(Radian) 단위의 입력값에 대해 사인, 코사인, 탄젠트 값을 계산하는 연산
   - 주기적 현상과 회전, 위치 계산을 수학적으로 표현 가능
   - 파동, 진동, 빛, 소리처럼 반복되는 현상을 모델링하거나 그래픽스에서 물체를 회전하거나 애니메이션을 표현할 때 필수 개념
   ```bash
     # np.sin(): 사인 값 계산
     np.sin(angles)

     # np.cos(): 코사인 값 계산
     np.cos(angles)

     # np.tan(): 탄젠트 값 계산
     np.tan(angles)
   ```
5. 지수 및 로그 연산 (Exponential and Logarithmic Functions)
   - 미분: 변화율 / 적분: 누적량
   - 부정적분(원시함수): 특정 구간을 지정하지 않고, 주어진 함수의 적분 일반형을 구하는
   - 데이터 규모가 크거나 변화가 급격할 때 값을 변환하여 안정적으로 다룰 수 있음
   ```bash
      # np.exp(): 지수 함수(e^x) 계산
      np.exp(values)
      
      # np.log(): 자연로그(밑 e) 계산
      np.log(values)
      
      # np.log10(): 상용로그(밑 10) 계산
      np.log10(values)
   ```
6. 브로드캐스팅과 유니버셜 함수
    - 반복문을 작성하지 않고도 스칼라와 배열, 크기가 다른 배열 간 연산을 간단히 표현 가능
    - 불필요한 메모리 복제를 줄이고 연산 속도를 높일 수 있음
    ```bash
      # np.add(): 배열과 스칼라 간 덧셈, 브로드캐스팅 적용
      np.add(array, scalar)
    ```
7. `out` 매개변수를 사용한 연산 결과 저장
   - 새 배열을 만들지 않고 지정한 배열에 결과를 저장함
   - 메모리 사용량 감소: 중간 결과용 임시 배열 생성을 피함
   - 할당 비용 절감: 대용량 연산에서 불필요한 메모리 할당을 줄임
   ```bash
     # np.multiply(): 요소별 곱, out으로 결과 저장
     np.multiply(a, 10, out=result)
   ```

### 2. Pandas

> 📢 구조화된 데이터의 조작과 분석을 위한 데이터프레임 및 시리즈 객체를 제공하는 파이썬 라이브러리 📢
>   - 다양한 형태의 데이터를 쉽게 조작하고 변환 가능

>   - 데이터를 불러오고 저장하는 작업을 간편하게 수행 가능

| 특징 | 설명 |
|---|---|
| 데이터 조작 및 변환 | Pandas는 행과 열을 기준으로 데이터를 선택, 필터링, 정렬, 집계 및 피벗할 수 있는 다양한 기능 제공<br>결측치 처리, 중복 제거, 데이터 형 변환 등을 통해 데이터를 정제하고 분석할 수 있음 |

### 2-1. 시리즈(Series)

> 📢 인덱스를 가지는 1차원 배열 형태의 데이터 구조 📢
>   - 한 개의 열(column)으로 구성된 데이터
>   - Pandas 데이터프레임의 구성 요소로 활용됨

<img width="640" height="258" alt="image" src="https://github.com/user-attachments/assets/b00d66c8-6518-4c49-a6d6-a30a77a04b33" />


| 특징 | 설명 |
|---|---|
| 인덱스를 기반으로 하는 1차원 구조 | - 1차원 데이터로 구성되며, 각 요소는 인덱스를 통해 접근이 가능함<br>- 기본적으로 정수형(0부터 시작) 인덱스를 가지지만, 특정한 레이블(문자열, 날짜 등)로 지정할 수도 있음 |
| 다양한 데이터 유형 지원 | - 정수, 실수, 문자열, 불리언, 날짜 데이터 등 다양한 유형의 데이터를 저장 가능<br>- 단일 데이터 타입을 가지지만, `dtype=object`를 통해 여러 유형을 함께 저장도 가능 |
| 유연한 데이터 생성 및 변환 | - 리스트, 딕셔너리, NumPy 배열 등을 사용하여 쉽게 Series를 생성 가능<br>- 필요한 경우 기존 데이터의 일부를 선택하여 새로운 Series로 변환 가능 |
| 벡터 연산 지원 | - NumPy와 유사한 벡터 연산이 가능하여, 반복문 없이 빠르게 연산을 수행가능<br>- 요소별 연산이 가능하며, 브로드캐스팅 기능 지원 |
| 누락된 데이터 처리 | - Series는 결측값(`NaN`)을 자동으로 처리하며, 누락된 데이터를 탐지하고 조작하는 다양한 기능을 제공 |

#### Pandas Series의 기본 속성
| 속성 | 설명 | 예제 |
|---|---|---|
| `values` | Series의 데이터 값을 ndarray 형식으로 반환 | `series.values` |
| `index` | Series의 인덱스를 반환 | `series.index` |
| `dtype` | Series의 데이터 유형을 반환 | `series.dtype` |
| `shape` | Series의 크기(길이)를 반환 | `series.shape` |
| `size` | Series의 총 요소 수 반환 | `series.size` |
| `name` | Series 객체의 이름 설정 및 확인 | `series.name = '이름'` |


> 📢 **사용 이유**: 데이터를 인덱스를 기준으로 구조화하여 원하는 데이터를 빠르게 조회하고, 연산 및 분석을 간결하게 수행할 수 있도록 하기 위해
>   - 인덱스를 활용한 데이터 접근성, 벡터 연산의 효율성, 결측치 처리 등의 기능을 통해 데이터 분석 프로세스를 최적화
>   - 인덱스를 활용한 데이터 저장과 조작
>   - 빠른 연산 및 벡터화 처리
>   - 데이터의 정합성 및 결측치(`NaN`) 처리
>   - 다양한 데이터 유형 지원
>   - CSV, JSON, SQL 등 다양한 데이터 소스와의 호환성

### 2-2. 데이터프레임(DataFrame)

> 📢 행과 열로 구성된 2차원 테이블 형태의 데이터 구조 📢

| 구성 요소 | 설명 |
|---|---|
| 데이터(values) | - DataFrame의 핵심 구성 요소로, 실제 데이터가 포함된 부분<br>- 각 열(column)은 개별적인 데이터 타입을 가질 수 있음 |
| 열(column, Series로 구성) | - DataFrame의 열은 Pandas의 Series 객체로 표현되며, 동일한 데이터 타입 유지함<br>- 각 열에는 이름(레이블)이 할당되어 데이터를 명확하게 식별 가능 |
| 행(row) | - DataFrame의 각 행은 서로 다른 속성의 데이터를 포함하며, 고유한 인덱스로 접근 가능<br>- 행 단위의 조작 및 필터링이 가능하여 데이터를 효과적으로 가공 가능 |
| 인덱스(index) | - 각 행을 고유하게 식별하는 레이블(기본적으로 0부터 시작)을 가짐<br>- 필요에 따라 문자열이나 날짜 형식의 인덱스를 사용 가능 |

> 📢 **사용 이유**: 데이터를 행과 열 형태로 정리하여 조회/수정/분석을 쉽게 수행할 수 있도록 하기 위해
>   - 행과 열 기반의 데이터 관리
>   - 벡터화 연산을 통한 빠른 데이터 처리 성능
>   - CSV, Excel, JSON, SQL 등 다양한 데이터 소스와의 호환성
>   - 결측치 및 이상값 처리 지원 -> 데이터 품질 유지
>   - 문자열 처리, 날짜 변환, 데이터 타입 변경, 중복 제거 등 데이터 정제 및 변환 용이

#### 사용 방법
1. 딕셔너리를 이용한 생성
   - 열(column) 이름과 값(value)을 정의해 직관적인 방식으로 데이터프레임 생성 가능
  ```bash
    import pandas as pd
    import numpy as np
    
    data = {'이름': ['홍길동', '김철수', '박영희'],
            '나이': [25, 30, 28],
            '성별': ['남', '남', '여']}
    
    df = pd.DataFrame(data)
    print(df)
  ```
  ```text
        이름  나이 성별
    0  홍길동  25  남
    1  김철수  30  남
    2  박영희  28  여
  ```
2. 리스트를 이용한 생성
   - 행 단위의 데이터를 입력하고, `columns` 매개변수를 통해 열 이름 지정 가능
  ```bash
    data = [[1, 2, 3],
            [4, 5, 6],
            [7, 8, 9]]
    
    df = pd.DataFrame(data, columns=['A', 'B', 'C'])
    print(df)
  ```
  ```text
       A  B  C
    0  1  2  3
    1  4  5  6
    2  7  8  9
  ```
3. NumPy 배열을 이용한 생성
   - 대규모 데이터 수치 데이터를 효과적으로 처리 가능
  ```bash
    data = np.array([[10, 20, 30],
                     [40, 50, 60]])
    
    df = pd.DataFrame(data, columns=['X', 'Y', 'Z'])
    print(df)
  ```
  ```text
        X   Y   Z
    0  10  20  30
    1  40  50  60
  ```

#### ❓ NumPy 데이터를 DataFrame으로 바꾸는 이유
- NumPy 배열은 수치 계산에는 강력하지만, 데이터에 이름을 붙이거나 결측치를 다루는 기능은 제한적
- Pandas DataFrame은 열 이름과 행 인덱스를 붙여서 데이터를 표 형태로 관리할 수 있음 & 결측치 처리, 그룹화, 통계 요약, 시계열 분석 같은 데이터 분석 기능 내장
- NumPy는 계산 중심 / Pandas DataFrame은 분석과 관리 중심
